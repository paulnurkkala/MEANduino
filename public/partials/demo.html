What is this?

One day, I decided that it would be interesting to take my Arduino and gather temperature information around my house, just to see what I could do with the data. I don't have particularly interesting ideas, but I thought that it would be worth attempting something that I had never done before and see if I can clearly present the data that I gather online using a MEAN stack.

The first thing I did was build the Arduino board, and put it in a "protective" case, because I live in Indiana, and there's snow this time of the year, which doesn't get along with electronics very well.

Thai food takeout boxes work pretty well!

Next, I hung the package out the window to get a little bit of distance away from the warmth of the insisde, just in case that skews the data.

About once a minute, the Arduino takes a measurement and sends it off to the MEAN server. The MEAN server that is listening for this GET request takes the information, saves it, and then allows the user to come back and query the information at their lesiure. In this case, I have it using a website called plot.ly to generate the information.

All of the code necessary to make this service happen is included in the github account. You can push these onto a server for free, with very little effort. Soon, you'll have your Arduino creation running online.
